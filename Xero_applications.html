# Xero Reset - 서비스 날짜 자동 갱신 [v5]
- Weekly / Fortnightly 줄의 날짜 태그(예: 06,20/06)를 다음 인보이스 달로 갱신
- 마지막 날짜 + 주기(매주 7일 / 격주 14일) -> 그 달에 들어오는 날짜만 수집
  - 예: 06,20/06 (격주) -> 04,18/07   /   06,13,20,27/06 (매주) -> 04,11,18,25/07
- (X) 취소표시 · (x2) 횟수표시는 무시, 날짜만 깔끔하게 (예: 05(X),09,19/06 -> 03,17,31/07)
  - 개수 · 취소 · 했는지 여부는 직원이 나중에 따로 수정
- 횟수가 바뀌면 Qty + 금액 + GST 도 자동 갱신 (격주 3번, 매주 5번 등)
- 점선 아래 정기 extra (예: rxCleaning bedsheets, 자체 날짜 태그 + Qty>0) = 바로 위 정기 청소 줄의 날짜를 복사
- 이미 다음 달이거나, Weekly/Fortnightly 도 아니고 위 정기 줄도 없으면 건너뜀

# Xero Reset - 점선 아래 (엑스트라 정리)
- Next invoice date 와 같은 달 = 그대로 둠
- 별표 개수(* 또는 **)와 무관하게 날짜로 판단 (** 도 지난달이면 삭제 대상)
- 다른 달이고 * 1개 (지난달 엑스트라, 설명 안에 날짜 있음):
  - toilet paper / hand towel / hand soap / dishwashing liquid / dishwasher tablet·detergent = 줄은 두고 $0 으로 (날짜 삭제)
  - rx 계정 (rxCleaning 등 = 정기 엑스트라) = 줄은 두고 $0 으로 (날짜 삭제)
  - 나머지 (toilet seat, toilet bins, toilet brush, rubbish, exCleaning, nCleaning 등) = 줄 삭제
- 지난 달 *Commenced 메모(금액/월요금 없는 것), 기간 지난 *(until July) 메모 = 줄 삭제
- Qty 0 이고 날짜 태그 없는 dormant 줄 ((Only when required) windows 등) = 그대로 둠
- 저장은 수동(Save 버튼)

# 범위 태그 (NOTICE 줄의 Item 칸) where applicable
- /w = 워시룸 줄만
- /b = 빈(Bins) 줄만
- 태그 없으면 = 전체 (또는 "OO only", Washroom 단어로 범위 지정)
- /w, /b 를 같이 둬도 각각 자기 줄만 적용. Cleaning 은 절대 안 건드림.

# 인상 방식
- % (예: CPI 3.5%) -> per service/week/fortnight 가격에만 % 적용, monthly 는 거기서 재계산 (weekly ×52/12, fortnightly ×26/12) 이라 오차 없음
- % 대신 새 가격을 직접 지정해도 됨 (예: 노티스에 "Bins 66.50" -> 그 줄을 $66.50 으로 설정)

# OK 누르면
- 범위(/w, /b 등): NOTICE 줄만 삭제, 날짜(1025) 는 그대로
- 전체 노티스(CPI / new rate / revised): NOTICE + 날짜 태그 삭제
- 저장은 수동(Save 버튼)

# Xero Reset - auto-refresh service dates [v5]
- Rolls each Weekly / Fortnightly date tag (e.g. 06,20/06) forward to the next invoice month
- Step from the last date by the cycle (weekly 7 / fortnightly 14 days), keep the dates landing in that month
  - e.g. 06,20/06 (fortnightly) -> 04,18/07   /   06,13,20,27/06 (weekly) -> 04,11,18,25/07
- (X) cancel marks and (x2) count marks are ignored - dates only (e.g. 05(X),09,19/06 -> 03,17,31/07)
  - counts / cancellations / done-or-not are fixed by staff later
- If the count changes, Qty + amount + GST update too (3 fortnightly, 5 weekly, etc.)
- Recurring extras below the divider (e.g. rxCleaning bedsheets, own date tag + Qty>0) = copy the dates from the regular cleaning line right above
- Skips lines already on the next month, or with no freq word and no regular line above

# Xero Reset - below the divider (clear extras)
- Same month as next invoice date = leave
- Judged by DATE regardless of asterisk count (* or **); a past-month ** is deleted too
- Different month and a single * (last month's extra, date inside the description):
  - toilet paper / hand towel / hand soap / dishwashing liquid / dishwasher tablet·detergent = keep line, set $0 (strip date)
  - rx account (rxCleaning etc. = recurring extra) = keep line, set $0 (strip date)
  - everything else (toilet seat, bins, brush, rubbish, exCleaning, nCleaning, etc.) = delete line
- past-month *Commenced memo lines (no amount/monthly rate) and expired *(until July) memo lines = delete
- Qty 0 dormant lines with no date tag ((Only when required) windows etc.) = leave
- Save manually (Save button)

# Scope tags (Item cell of the NOTICE line) where applicable
- /w = washroom lines only
- /b = bin lines only
- no tag = all (or scoped by "XX only" / the word Washroom)
- /w and /b together each apply to their own lines. Cleaning is never touched.

# Increase method
- % (e.g. CPI 3.5%) -> applies only to the per service/week/fortnight rate; monthly is recomputed from it (weekly ×52/12, fortnightly ×26/12) so there is no rounding drift
- or set a new price directly (e.g. notice says "Bins 66.50" -> set that line to $66.50)

# On OK
- scoped (/w, /b etc.): remove the NOTICE line only, keep the date tags (1025)
- full notice (CPI / new rate / revised): remove NOTICE + date tags
- Save manually (Save button)
